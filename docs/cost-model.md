# 成本計算模型

> 基於 Series 1 和 Series 3 實驗數據，計算 gstack-plus 路由策略的實際成本節省。

---

## 1. 成本計算公式

```
路由後成本 = Σ (任務i × 對應 Tier 成本/token)

理論節省率 = (All-Tier-A 成本 - 路由後成本) / All-Tier-A 成本

實際節省 = 理論節省率 × 月均 AI 任務預算
```

---

## 2. 三個 Tier 的成本基準（2026-05 數據）

| Tier | 典型模型 | 輸入 $/1M tokens | 輸出 $/1M tokens | 典型任務成本 |
|------|---------|-----------------|-----------------|------------|
| Tier-A | Opus 4.7 | $15 | $75 | $0.045–$0.08 |
| Tier-Mid | Sonnet 4.6 | $3 | $15 | $0.006–$0.015 |
| Tier-Exec | Haiku / Qwen | $0.08–$0.25 | $0.30–$1.25 | $0.00014–$0.002 |

---

## 3. 基於任務分佈的節省估算

Series 3 真實 git commit 分佈：45% Exec + 35% Mid + 20% Tier-A

| 分佈場景 | 預計節省率 | 適用場景 |
|---------|----------|---------|
| 均衡（45%/35%/20%） | **−46%** | 典型中型項目 |
| 重執行（60%/30%/10%） | **−65%** | 維護期項目 |
| 重架構（10%/30%/60%） | **−12%** | 早期設計期 |
| Exec 主導（70%/20%/10%） | **−75%** | DevOps/腳本為主 |

---

## 4. 計算器模板

```
輸入：
- 每月 AI 任務總數：N
- 任務分佈：E% Exec + M% Mid + A% Tier-A
- 當前模型：Opus（或 Sonnet）

輸出：
- 當前月均成本估算
- 路由後月均成本估算
- 節省金額和百分比
```

### 計算示例

假設：
- 每月 200 個任務
- 分佈：45% Exec + 35% Mid + 20% Tier-A
- 當前全部使用 Opus

**當前成本**（全部 Opus）：
200 × $0.060/任務 = $12.00/月

**路由後成本**：
- Exec: 200 × 45% × $0.001 = $0.09
- Mid: 200 × 35% × $0.010 = $0.70
- Tier-A: 200 × 20% × $0.060 = $2.40
- **總計：$3.19/月**

**節省**：$12.00 - $3.19 = **$8.81/月（−73%）**

## 5. 正確 Prompt 的額外節省

Series 5 發現：使用 S1 提示詞策略（角色 + 深度思考指令），Sonnet 達到 15.0/15，超越 Opus 的 12.7/15，成本只有 Opus 的 13.3%。

這意味著在 Tier-Mid 任務上，不僅路由本身節省 83%，正確使用提示詞策略還能進一步提升品質。
